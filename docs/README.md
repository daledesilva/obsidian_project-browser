# Project Browser documentation

This folder contains documentation for the Project Browser Obsidian plugin.

If you’re new to the codebase or the plugin’s mental model, start with **Overview**.

## Start here

- [Overview](overview.md): what the plugin is, the three main surfaces, and the end-to-end flow.

## Core concepts

- [Card Browser and navigation](card-browser-and-navigation.md): the “always in a folder” browsing model, sections, navigation, and search.
- [States and sections](states-and-sections.md): visible vs hidden states, No status, and how states drive grouping.

## Features

- [Projects](projects.md): folders that behave like cards, and how project status works.
- [Project Pages FAB](project-pages-fab.md): the in-editor floating action button for jumping between project pages.
- [File type visibility](file-type-visibility.md): control what file types appear in the Card Browser and in the pages menu.
- [Settings](settings.md): access/behaviour and display settings that change how the plugin feels.

## Contributing / maintenance

- [Testing](testing.md): how to run unit and E2E tests, and how they’re structured.
- [Known issues and attempted fixes](known-issues-and-attempted-fixes.md): limitations and past investigations (useful when debugging).

## Quick paths by task

- **Understand the product**: Overview → Card Browser and navigation → States and sections
- **Configure what you see**: Settings → File type visibility
- **Understand “projects”**: Projects → Project Pages FAB
